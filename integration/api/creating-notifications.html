      Creating notifications

      These section describes how you can send your first notification using curl calls. Notifir provides REST API and GraphQL API to make a request. The new notification will be shown in the notification inbox of the recipient in real-time.

      All requests require following headers:

      Headerâ€ƒâ€ƒâ€ƒDescription
      x-user-idcontaining the id of the user performing the request
      x-api-keycontaining project's API key
      x-user-hmaccontaining the computed hash for the user (required in case of hmac-based authentication)

      REST API#

      POST /api/notifications

      Example usage:

      curl --request POST \
     --url https://t-notifications-api.herokuapp.com/api/notifications \
     --header 'Content-Type: application/json' \
     --header 'x-api-key: [api_key]' \
     --header 'x-user-id: [user_id]' \
     --header 'x-user-hmac: [user_id_hmac]' \
     --data '
       {
         "type": "entry-created",
         "user_id": "beast~mailinator.com@marvel-x-men",
         "project_id": "default",
         "payload": {
            "entryTitle": "001",
            "stepTitle": "New",
            "folderTitle": "Reports",
            "user": "Henry McCoy"
         }
       }'

      GraphQL API#

      POST /api/graphql

      Example usage:

      curl --request POST \
     --url https://t-notifications-api.herokuapp.com/api/graphql \
     --header 'Content-Type: application/json' \
     --header 'x-api-key: [api_key]' \
     --header 'x-user-id: [user_id]' \
     --header 'x-user-hmac: [user_id_hmac]' \
     --data '
       {
         "query": "mutation { createNotification(input: { notification: { type: \"entry-created\", userId: \"beast~mailinator.com@marvel-x-men\", projectId: \"default\", payload: \"{\\\"entryTitle\\\": \\\"001\\\", \\\"stepTitle\\\": \\\"New\\\", \\\"folderTitle\\\": \\\"Reports\\\", \\\"user\\\": \\\"Henry McCoy\\\" }\" } } ) { notification { id } } }"
       }'
