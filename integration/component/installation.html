      Installation

      You can install wc-notification-bell component in different ways depending on your technology stack.

      Using script tag#

      <script
        type="module"
        src="https://cdn.jsdelivr.net/npm/wc-notification-bell@latest"
      />
      ...
      <notification-bell
        apiUrl="https://t-notifications-api.herokuapp.com/api/graphql"
        apiKey="114ee1da-067b-11ed-be0f-6f24634ae754"
        userId="beast~mailinator.com@marvel-x-men"
        userHmac="Cq911/p2a5MCmnobRhnjOEBwSm/RwFvq+IMGtZDw/hk="
      />

      Using npm#

      npm install wc-notification-bell --save

      And then you need to import the module before you can use it in your html/jsx/template/whatever:

      import "wc-notification-bell";
      ...
      <notification-bell mock />

      Configuration Parameters#

      ParameterTypeDescription
      mockbooleanAllows to play with widget without having API run (default: false)
      apiUrlstringThe URL of the Notifications API (GraphQL) to fetch notifications
      apiKeystringThe public API key of your project, which is safe to store on the client-side and share over the network
      userIdstringThe user id, which is usually email (but not required to be an email)
      userHmacstringThe HMAC code of the user's id, which is generated on the backend using the apiSecretKey and used for authentication
      localestringThe locale of the component (default: en_GB)

      Browser Support#

      This web component uses HTML templates, the shadow DOM and custom elements. If you need to polyfill for any of these standards then take a look at the web components polyfill.
