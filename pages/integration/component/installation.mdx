# Installation

You can install `wc-notification-bell` component in different ways depending on your technology stack.

## Using `script` tag

```html
<script
  type="module"
  src="https://cdn.jsdelivr.net/npm/wc-notification-bell@latest"
/>

...

<notification-bell
  apiUrl="https://t-notifications-api.herokuapp.com/api/graphql"
  apiKey="114ee1da-067b-11ed-be0f-6f24634ae754"
  userId="beast~mailinator.com@marvel-x-men"
  userHmac="Cq911/p2a5MCmnobRhnjOEBwSm/RwFvq+IMGtZDw/hk="
/>
```

## Using `npm`

```shell
npm install wc-notification-bell --save
```

And then you need to import the module before you can use it in your html/jsx/template/whatever:

```js
import "wc-notification-bell";

...

<notification-bell mock />
```

## Configuration Parameters

| Parameter |  Type   | Description                                                                                                          |
| :-------- | :-----: | :------------------------------------------------------------------------------------------------------------------- |
| mock      | boolean | Allows to play with widget without having API run (default: `false`)                                                 |
| apiUrl    | string  | The URL of the Notifications API (GraphQL) to fetch notifications                                                    |
| apiKey    | string  | The public API key of your project, which is safe to store on the client-side and share over the network             |
| userId    | string  | The user id, which is usually email (but not required to be an email)                                                |
| userHmac  | string  | The HMAC code of the user's id, which is generated on the backend using the apiSecretKey and used for authentication |
| locale    | string  | The locale of the component (default: `en_GB`)                                                                       |
